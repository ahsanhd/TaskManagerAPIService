# What I Learned From Building This Project

This project taught me the core mechanics of a real backend in code.

## Main Code Skills Learned

- how to create an Express app and start a server
- how to split backend code into routes, controllers, middleware, utilities, and config files
- how to define and use a database schema with Prisma
- how to read and write data through an ORM instead of writing raw SQL
- how to validate request input before it reaches business logic
- how to hash passwords instead of storing plain text
- how to generate and verify JWT tokens
- how to protect routes with authentication middleware
- how to enforce ownership rules so one user cannot access another user's data
- how to handle errors in one central place
- how to write integration tests that prove the API behaves correctly

## What This Taught Me About Backend Structure

- how an Express request flows through the app
- how backend layers are separated
- how a database-backed API is structured
- how authentication and authorization work in practice
- how to keep backend code maintainable and testable

## Short Version

I did not just build an app. I learned the skeleton of a backend service and how the pieces fit together.

That includes server setup, routing, middleware, controllers, Prisma, SQL modeling, auth, validation, error handling, and testing.
